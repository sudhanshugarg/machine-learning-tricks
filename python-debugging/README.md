# Python Debugging

This folder contains practice problems focused on Python debugging sessions.

## Goal

Collect debugging-oriented problems where the task is to identify, reproduce, and fix bugs in Python code.

## Interview Standard

- You will debug and fix multiple issues in a Python command-line application.
- The interview emphasizes code quality, core coding fundamentals, and speed.
- We also evaluate how effectively you use your coding knowledge to take smart shortcuts and solve problems efficiently.
- AI tools and coding assistants are not allowed.
- Total duration: 1 hour.

## Suggested Structure

Each problem can live in its own subfolder, for example:

- `problem_name/README.md` for the prompt and debugging context
- `problem_name/buggy.py` for intentionally buggy code
- `problem_name/tofix.py` for intentionally buggy code without any comments so that the candidate can begin here.
- `problem_name/tests.py` for validation
- `problem_name/solution.py` for a fixed version
